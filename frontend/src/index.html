Configure Azure Credentials

Enter your Azure Service Principal credentials. These are stored securely in AWS Secrets Manager.

Step 1: Create a Service Principal

Run these commands in Azure CLI to create a Service Principal with the required permissions:

az login
az ad sp create-for-rbac --name "CUDly" --role "Reservation Administrator" --scopes /subscriptions/<your-subscription-id>

The command output will contain the Tenant ID, Client ID (appId), and Client Secret (password).

Step 2: Enter the credentials below

Or configure via CLI:

cudly configure-azure --profile <your-profile>

Configure GCP Credentials

Paste your GCP Service Account JSON key. These are stored securely in AWS Secrets Manager.

Step 1: Create a Service Account

Run these commands in gcloud CLI to create a Service Account with the required permissions:

gcloud auth login
gcloud iam service-accounts create cudly --display-name="CUDly Commitment Manager"
gcloud projects add-iam-policy-binding <PROJECT_ID> --member="serviceAccount:cudly@<PROJECT_ID>.iam.gserviceaccount.com" --role="roles/compute.admin"
gcloud iam service-accounts keys create ~/cudly-key.json --iam-account=cudly@<PROJECT_ID>.iam.gserviceaccount.com

Then open ~/cudly-key.json and paste its contents below.

Step 2: Paste the JSON key below

Or configure via CLI:

cudly configure-gcp --profile <your-profile>
